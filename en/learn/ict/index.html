I have choosen to split this into three aspects: Hardware, Networks and Software. Hardware CPU Networks Network Protocols TCP/IP

TCP/IP is not a protocol. It is a collection of protocols named after the two most important ones in it.

The TCP layer handles the message. The message is broken down into packets which is sent over the internet. The packets are received by the receiving computer and are reassembled into the original by it’s TCP layer.

The IP layer is primarily used to transmit the message. This is done by assigning an IP address to every device in the network.

How email works:
  1. The sender writes and email using an email client on their computer such as Gmail or Yahoo
  2. When the email is sent, the email is uploaded to the SMTP (Simple Mail Transfer Protocol) server as outgoing mail.
  3. All messages wait in a queue while the server communicates with the DNS (Domain Name Server) to find where to send the email.
    1. If the address can not be found, sender will receive a mail failure message. OR
    2. If the address is found, the message and attachments are transferred.
POP & IMAP

POP (Post Office Protocol) allows all messages on the server to be downloaded to the device so that the server does not run out of space. However, that means that it is hard to view the same email on different devices. Also if you delete a message on your computer, it may not delete on your phone.

IMAP however is bidirectional so all messages are synchronised on all logged in devices because it is stored on the server. This is perfect for clients, but this method means that email providers may need many servers to handle requests and store the emails.
